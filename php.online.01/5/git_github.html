Ejercicio 5

Git y GitHub

Sistemas de Control de Versiones

Joint Developer define el control de versiones como un sistema que registra los cambios realizados sobre un archivo o conjunto de archivos a lo largo del tiempo de tal manera que sea posible recuperar versiones especificas más adelante.

Los Sistemas de control de versiones (en adelante, SCV) se pueden clasificar en tres tipos: locales, centralizados y distribuidos.

SCV Locales

Los primeros SCV almacenaban cada fase o estado de las modificaciones como versiones en una base de datos local. Si se quería recuperar una versión anterior, el SCV actúa de intermediario a la hora de gestionar los archivos, evitando que se puedan borrar por error.

SCV Centralizados

Los sistemas locales tenían la dificultad de hacer difícil compartir código entre varias personas. Para solucionar esto los SCV almacenan las versiones en un servidor, de modo que están disponibles para varios usuarios.

Con el aumento de los usuarios surge el problema de cómo gestionar múltiples ediciones de un mismo archivo. Los sistemas centralizados recurren a la figura de un tercero para resolver de forma manual los conflictos entre versiones diferentes, además de prohibir que un usuario elimine o sobreescriba el trabajo de otro.

SCV Distribuidos

En proyectos muy grandes y activos, la solución anterior se vuelve muy engorrosa. La solución fue dar a cada usuario una copia local de cada proyecto, creando una red de repositorios.

Un repositorio es un paquete que contiene archivos digitales, en su mayor parte scripts pero también puede contener cualquier otro tipo de archivo.

Git y Github

Git es un SCV distribuido diseñado por Linus Torvalds, el creador del sistema operativo Linux, en 2005, y a día de hoy es una de las herramientas de control de versiones más utilizadas en el mundo, si no la que más.

Sobre la funcionalidad de Git se han construido muchos proyectos, algunos centrados en ampliar sus capacidades y animar a la comunidad de desarrolladores a compartir su código y colaborar. Este es el caso de GitHub, una plataforma donde cualquiera puede ver, compartir, clonar y editar sus propios repositorios o los de otros usuarios.

Diferencias principales

Git es un programa local para control de versiones; ni más ni menos. Su interfaz es muy básica, mediante línea de comandos, aunque algunas aplicaciones hacen su uso más visual, como Tig(sin salir del modo texto), u otros más completos y con interfaz gráfica, como GitKraken.

Sin ser una red social, GitHub comparte algunas de sus características, como ofrecer una página de perfil para cada usuario, páginas personales y opciones para seguir o marcar como favoritos a otros usuarios o alguno de sus repositorios. En particular, las páginas de usuario se han convertido en una herramienta muy útil para buscar empleo a través de las páginas de perfil, donde un futuro empleador puede ver de primera mano nuestros proyectos y habilidades.

La diferencia principal entre los dos es que Git es un herramienta instalada de forma local, mientras que GitHub es un servicio online para manejar y compartir repositorios creados con Git.
