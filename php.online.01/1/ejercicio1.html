FrontEnd y BackEnd

Diferencias entre front-end, back-end y full-stack

Los términos front-end, back-end y full-stack se usan para clasificar a los desarrolladores web según el campo en el que se especializan.

  • Front-end: se centran en implementar el aspecto visual de la aplicación y todo lo relacionado con la interfaz y experiencia del usuario (UI y UX). Un desarrollador front-end maneja HTML, CSS y JavaScript, fundamentalmente.
  • Back-end: controlan el buen funcionamiento de la aplicación en el lado del servidor, gestionando el procesado y comunicación de los datos entre la aplicación y una base de datos. Deben conocer por lo tanto, lenguajes de servidor (PHP, Java, Perl o JavaScript), de base de datos (variantes de SQL si se emplea una base de datos relacional, o MongoDB, por ejemplo, si se emplea otro tipo de base de datos).
  • Full-stack: es la combinación de los dos anteriores. En teoría, un desarrollador full-stack es el que destaca en ambos lados de una aplicación web: cliente y servidor.

¿Qué es WCAG?

Son las siglas de Web Content Accesibility Guidelines (Pautas de Accesibilidad para el Contenido Web), desarrolladas por la W3C en cooperación con personas y organizaciones de todo el mundo, con el objetivo de hacer la web más fácil fundamentalmente para personas con discapacidad, pero también para cualquier agente de usuario, incluidos dispositivos con capacidades reducidas, como ciertos móviles.

A día de hoy la versión más reciente es la WCAG 2.1. La versión 2.0 fue publicada en diciembre de 2008, y convertida en un estándar ISO en 2012.

Estas pautas o directrices se asientan en cuatro principios:

  1. Perceptibilidad: se recomienda proporcionar alternativas a cualquier contenido de naturaleza no textual, como descripciones o símbolos. El contenido textual que sea complejo por sí mismo o por su representación visual debe poder presentarse de forma más sencilla y fácil de comprender.
  2. Operabilidad: la interfaz de usuario debe ser intuitiva, indicando al usuario donde se encuentra en todo momento y qué opciones tiene. Si es necesario debe contener sistemas de ayuda. No debe imponer limitaciones de tiempo y, si es posible, debe poder ser controlada desde teclado.
  3. Inteligibilidad: se aplica tanto al contenido como a la interfaz de usuario; se recomienda el uso de patrones visuales y de interacción predecibles, así como implementar ayudas para corregir al usuario en caso de error.
  4. Robustez: el contenido web debe poder ser interpretado correctamente por el mayor número posible de agentes de usuario.

Importancia de validar el código

Cuando hablamos de validación nos referimos al proceso porque el que se comprueba que algo tiene un formato determinado y cumple con ciertas restricciones. En el caso de una página HTML, decimos que está validada cuando todas las etiquetas están cerradas y no hay ninguna solapada (todas las etiquetas terminan antes de que se abra otra o se cierre la que la contiene).

Hay varias razones por las que se recomienda asegurarnos de que nuestro código cumpla con los requisitos necesarios, y es especialmente importante en el caso de que manejemos datos introducidos por el usuario.

Validación de HTML

Aporta las siguientes ventajas:

  • Compatibilidad: asegurar que nuestra aplicación sea utilizable en el mayor número posible de dispositivos.
  • Visualización: asegurar que el contenido de nuestra aplicación se vea correctamente.
  • Mantenimiento: un código validado cumple con unos estándares y por lo tanto tiene una estructura predecible y es más fácil de revisar en buscar de errores o de modificar para añadir mejoras.

Validación de Formularios

Es especialmente importante para garantizar el buen funcionamiento de la aplicación. En particular, ayuda a controlar los siguientes aspectos:

  • Datos correctos: garantiza que los datos introducidos tienen el formato correcto.
  • Privacidad: proteger los datos de los usuarios para que sean manipulados o lleguen a manos de terceros.
  • Seguridad: proteger la propia aplicación de ataques que causen un mal funcionamiento o alteren la aplicación.
